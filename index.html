    1. KoNLPy 기반 작사 모델의 표현 한계
      • 문제 : KoNLPy 기반 작사 모델의 표현 한계
      • 원인 : 국립국어원 기준을 참고하여 분석된 형태소를 조합하는 형태로 문장을 생성하려 했으나, 형태소로 나뉜 데이터의 양이 너무 방대하고 원하는 형태의 문장이 나타내는것이 어렵다는것을 느낌
      • 해결 : 단순한 형태소 조합이 아닌, 문맥 기반 생성이 가능한 KoGPT2 모델로 전환

    2. KoGPT2 모델의 구성과 구조, 언어 모델의 파인튜닝 절차에 대한 이해 부족
      • 문제 : 입력 데이터 길이 초과로 인한 에러 발생 또는 모델 성능 저하
      • 원인 : KoGPT2는 최대 토큰 길이가 존재함 (max_length=1024 등). 이를 초과하는 학습 데이터가 있을 경우 학습 시 에러 또는 성능 저하 발생
      • 해결 : tokenizer 사용 시 truncation=True, padding=True 옵션을 통해 길이 제어. 최대 길이를 기준으로 슬라이딩 윈도우 방식으로 잘라 학습셋 구성

    3. 하이퍼파라미터 설정 값, 학습률과 경사하강법에 대한 이해 부족
      • 문제 : 토크나이징 처리 미흡, 학습 텍스트의 다양성과 길이 부족, 적절한 학습 스케줄 부재
      • 원인 : 처음 수행하는 파인튜닝 작업이었기 때문에 토크나이저, 패딩, 옵티마이저, 스케줄러 설정 등 학습 부족
      • 해결 : 공부해가면서 여러 실험을 거쳐 적정값을 수동으로 튜닝하며, weight_decay 같은 안정화 요소도 고려해 설정
