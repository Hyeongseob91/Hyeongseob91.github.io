    Problem

    Business Challenge

    시성비(시간 대비 효율) 트렌드 - 영상이나 음성 콘텐츠에서 원하는 부분만 빠르게 취사선택하려는 소비자가 늘어남에 따라, 요약 서비스의 필요성이 대두되었습니다.

    Social Challenge

    시니어 금융 리터러시 - 5060 세대가 모바일 자산관리 및 금융 거래에 적극적으로 참여하고 있으나, 복잡한 정보를 습득하는 데 여전히 어려움을 겪고 있습니다.

    • 긴 영상 시청에 대한 부담

    Solution

    What We Built

    AI 기반 경제 유튜브 채널 요약 서비스

    클릭 한 번으로 영상을 요약 보고서로 변환하는 웹 애플리케이션입니다. Whisper(STT)와 Gemma 2(LLM)를 활용한 "음성 추출 → 텍스트 변환 → 요약" 파이프라인을 구축했습니다.

    Core Value

    • 효율적인 정보 전달: 경제 및 금융 영상의 내용을 분석하여 높은 효율의 문서화된 정보를 생성
    • 금융 지식 격차 해소: 정보 접근성이 낮은 타겟 사용자들이 경제 정보를 더 쉽게 접하도록 지원

    주요 성과

    • E2E 파이프라인 — 음성 추출 → STT → 요약까지 원클릭 자동화
    • STT 엔진 — Whisper-large-v3 한국어 음성 인식 적용
    • 모델 경량화 — Gemma-2-2b-it QLoRA 4-bit 양자화 적용
    • 백엔드 모듈화 — FastAPI 3개 서비스 라우터 (Audio, STT, Summary)
    • 파인튜닝 검증 — LoRA/QLoRA 한계 분석 및 베이스라인 비교 체계 구축

    Target Users

    • AI/금융 비전문가 (시니어층)
    • 바쁜 직장인 및 학생

    Architecture Overview

    • 오디오 추출: yt-dlp + FFmpeg (192kbps MP3)
    • 음성-텍스트 변환: OpenAI Whisper-large-v3 (잡음에 강하고 한국어 인식 정확도 높음)
    • 텍스트 요약: Google Gemma-2-2b-it (QLoRA 파인튜닝, 4-bit 양자화)
    • 백엔드: FastAPI (3개 라우터: Audio, STT, Summary)
    • 프론트엔드: Streamlit

    Model Fine-Tuning

    STT (Whisper)

    LoRA 기법으로 파인튜닝을 시도했으나, 에포크가 진행될수록 오차율(CER, WER)이 지속적으로 상승하여 성능이 저하되는 것을 확인했습니다. 이에 따라 원본 Whisper-large 모델을 그대로 사용하는 것이 더 낫다는 결론을 내렸습니다.

    LLM (Gemma 2)

    메모리 효율을 위해 QLoRA(양자화+LoRA) 기법을 사용하여 튜닝을 진행했습니다. 4-bit BitsAndBytes 양자화를 적용하여 VRAM 사용량은 줄었으나, 오히려 요약 품질이 저하되어 튜닝하지 않은 원본 모델보다 못한 결과가 나왔습니다. 경량화와 성능 사이의 트레이드오프를 체감한 경험이었습니다.

    성과 및 회고

    주요 성과

    • yt-dlp + FFmpeg 기반 오디오 자동 추출 파이프라인 구축
    • FastAPI 백엔드 + Streamlit 프론트엔드 통합
    • QLoRA 적용으로 VRAM 효율화 (4-bit 양자화)

    시행착오

    • STT LoRA 파인튜닝 실패: 파인튜닝 기술에 대한 이해 부족과 보유 인프라의 한계를 사전에 가늠하지 못한 채 진행하여 실패
    • LLM QLoRA 경량화 역효과: 4-bit 양자화로 경량화는 달성했으나, 오히려 성능이 저하되어 튜닝하지 않은 원본 모델보다 못한 결과 도출
    • 긴 텍스트 문맥 누락: 청킹(Chunking) 전략의 필요성 인식
    • 동일 문장 중복 출력: 후처리 로직 추가로 해결

    기술적 성장

    처음 접하는 STT/LLM 파이프라인과 파인튜닝 기술이었기에, 접근 방법 자체가 잘못된 부분이 많았습니다. 그러나 이 경험을 통해 모델 튜닝 전 베이스라인 성능 측정의 중요성, 인프라 제약 조건 사전 파악, 그리고 양자화와 성능 간의 트레이드오프를 체감할 수 있었습니다. 실패를 통해 앞으로 어떤 방향으로 공부하고 나아가야 할지 명확해졌습니다.
