    Overview

    "The user only picks a goal — five AI experts cross-validate each other while debating, and return a data-backed Top 3 of districts. A HOTL (Human-Out-of-The-Loop) decision platform."

    Generic public-data dashboards answer descriptive questions ("how many people walk through Yeoksam-dong?") but not decision questions ("where should I open my cafe?"). WIGTN FLAKE starts from the user's goal, cross-analyzes four datasets (real estate, foot traffic, card sales, telecom contracts), and uses a Snowflake Cortex × GPT-4o hybrid orchestration so a panel of five AI experts cross-validates each other in real-time debate and returns Top-3 recommendations + auto-detected anomaly signals + a 6-month forecast.

    Problem — Why Existing Dashboards Don't Decide

    Goal-agnostic tools

    Foot traffic matters to advertisers but not to cafe owners; income data fits rental appliances but not billboards. The same metric needs different interpretations per goal to drive a decision — but standard dashboards present flat numbers detached from any goal.

    Single-perspective AI

    Off-the-shelf LLM helpers produce "confident-sounding paragraphs" — no debate, no contradicting expert, no SQL trail. Decisions need multi-perspective verification.

    Manual cross-dataset integration

    Real estate × foot traffic × card sales × telecom live in different tools. Wiring them up for a single decision can take weeks of spreadsheet work.

    Architecture — A 5-Stage Flow Visible to the User

    1. Purpose Selection — six preset cards (cafe · rental appliances · billboard · investment · anomaly detection · free-form) or free-form input
    2. District Context — AI suggests candidate districts; goal + districts form the brief
    3. Expert Orchestration — a GPT-4o summoner activates five purpose-tuned personas. Each expert cross-validates the others' reasoning, neutralizing single-LLM hallucination and bias through multi-perspective consensus. Live debate streams over SSE.
    4. Cortex Execution — the user's natural-language question is auto-translated to SQL by Cortex Analyst on top of the Semantic Models → live queries across four datasets (SPH · RichGo · NextTrade · AJD) + parallel ANOMALY_DETECTION and FORECAST → results converge in the warehouse.
    5. Decision Report — Top-3 ranking + anomaly badges + 6-month projections + a goal-specific action checklist

    Engineering Principles

    • HOTL (Human-Out-of-The-Loop) — the human only picks the goal; the decision-making process itself is owned by an autonomous loop of five AI experts debating and cross-validating each other. Trustworthy recommendations should fall out of the loop without per-step human intervention — single-model hallucination is offset by multi-perspective consensus.
    • Cross-validation as core value — "debate" is the form; the substance is verification. Five personas each bring a different slice of data, contest and reinforce each other's conclusions, and use the consensus mechanism to block the hallucinations and biases a single LLM might leak through.
    • Purpose-first design — every component, from UX to code, carries the goal context
    • Hybrid AI — Cortex for data / SQL / reports; GPT-4o for debate personas — best model per layer
    • Snowflake as lead role — 11 Cortex features woven through the demo, not checklisted
    • All four datasets live — wired to Semantic Models and queried during the demo
    • Demo-first approach — the 10-minute video promotes ANOMALY_DETECTION to the climax
    • Code reuse — ~90% of orchestrator / chat reused from WIGENT (Build with TRAE Seoul winner)
    • Honest engineering — slide claims match code; decisions are benchmark-backed
