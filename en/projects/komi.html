KOMI — AI Remote Rehabilitation Service

Wanted Learning Project Lead / Tech Lead (AI) Mar 2025 (1 month) FastAPI + WebSocket + RAG Pipeline
YOLO11 Pose Detection LangChain RAG ChromaDB WebSocket FastAPI Streamlit OpenCV

Contents

    Problem

    Social impact

    Aging populations and rising chronic illness are driving a surge in rehabilitation demand. But people with reduced mobility — and those in regions with limited medical access (rural areas, islands, lighthouses, military bases) — find it hard to visit clinics. Without a physical therapist's real-time feedback, exercising with correct form is difficult.

    Economic impact

    As Korea's national health insurance faces fiscal strain, there is growing need for a preventive health-management model rather than treatment-after-the-fact — low-cost, high-efficiency rehabilitation that lowers medical bills and helps the system stay solvent.

    Technical challenge

    - Real-time streaming: WebSocket plus multithreading to synchronize video and minimize latency.
    - RAG validation: Block hallucination in LLM-generated rehab feedback and ensure medical credibility through an objective verification framework.

    Solution

    What we built

    • YOLO11-based pose-estimation engine (real-time form detection)
    • Pose-evaluation dataset and algorithm (reference-pose comparison)
    • LangChain-based RAG system (medical literature retrieval + feedback generation)
    • WebSocket-based real-time communication (multi-camera sync)
    • RAGAS evaluation framework (RAG quality validation)

    Core value

    • Core concept: Multiple webcams analyze the user's movement in real time, and an AI provides personalized exercise feedback. YOLO11 pose detection plus LLM-driven feedback delivers tailored rehabilitation guidance anytime, anywhere.
    • Dual-view analysis: Front and side cameras together for 3D-aware analysis, capturing depth and joint angles a single view would miss.
    • Evidence-based feedback: Medical papers and rehab literature stored in a vector DB; the RAG pipeline produces clinically-grounded, personalized corrective feedback.

    Highlights

    • RAGAS Context Recall — 1.0 (100%)

    1. Overall workflow — from user input to feedback generation

    Service Flow

    Precision-analysis mode

    Pick exercise → study guide video → record → extract frames → YOLO11 pose detection → reference-pose comparison → per-joint accuracy → LLM feedback → result visualization

    Real-time analysis mode

    Pick exercise → connect webcam → Base64 encode → WebSocket stream → pose detection → score → instant feedback

    1. Main — exercise selection
    2. Guide — learn the correct form
    3. Precision analysis — segmentation-based recording & pose analysis
    4. Result — per-joint accuracy
    5. Result — LLM-driven improvement suggestions
    6. Real-time analysis — instant form feedback

    What I Built

    1. Pose-estimation engine

    A real-time pose-detection engine built around the YoloPoseModel class.

    • YOLO11n-based real-time pose detection
    • 17 COCO Keypoints: nose, eyes, ears, shoulders, elbows, wrists, hips, knees, ankles
    • OpenCV-based frame processing and keypoint visualization
    • Filter joints by confidence (conf_threshold=0.5)
    • SKELETON-array-driven joint-link visualization (arms, legs, torso)
    • Base64 ↔ NumPy array conversion

    2. Pose-evaluation dataset & algorithm

    Dataset of correct poses + evaluation algorithm in the PoseAnalyzer class.

    • Collected and cleaned reference-pose data
    • Mapped misaligned poses to root causes and remediation
    • 3-tier data structure: pose → cause → fix
    • Joint-angle calculation via dot product (_calculate_angle)
    • L2 distance + cosine similarity for similarity scoring
    • Joint flagged as misaligned if > 15° from reference

    3. LangChain RAG system

    Medical-literature retrieval + feedback generation built on LangChain + ChromaDB.

    • OpenAIEmbeddings() to vectorize medical PDFs
    • ChromaDB vector store + retrieval pipeline
    • retriever.as_retriever(search_type="similarity", k=5) similarity retrieval
    • Chain: RunnableMap → PromptTemplate → ChatOpenAI(gpt-4o-mini)
    • Per-joint error stats turned into natural-language prompts (generate_summary_prompt)
    • Prompt engineering from a clinician's point of view

    4. RAGAS evaluation framework

    Adopted RAGAS to objectively evaluate RAG quality.
    - "Is retrieval accurate?"
    - "Does the answer match the question?"
    - "Is the answer faithful to the evidence?"
    Quantitative answers to these questions gave us data-driven QA, not gut feel.

    Outcomes & Retrospective

    Key outcomes

    • YOLO11 + LangChain integrated rehabilitation service
    • Dual modes: precision analysis / real-time analysis
    • WebSocket-based multi-camera sync
    • RAG performance validated with RAGAS metrics

    RAGAS results

    • Context Precision: 1.0 (100%) — retrieved context highly relevant to the question
    • Context Recall: 1.0 (100%) — required information retrieved without omission
    • Answer Relevancy: 0.82 (82%) — generated answer appropriately matches the question
    • Faithfulness: 0.61 (61%) — identified as a clear improvement target; flags advanced prompt engineering as next step

    Lessons learned

    The hardest part was collecting medical data. Privacy concerns are significant, and data on AI-Hub also requires fees, so we couldn't keep the original plan as-is.
    We pivoted from rehab-exercise correction to a posture-correction project. After the pivot, another problem appeared: we had planned to identify causes for an inability to lift an arm and predict them. After consulting with an orthopedic specialist on stored data, we concluded that single-2D-camera analysis can't establish medical criteria — the field has too many edge cases.

    That made us redefine the problem to focus on whether the movement is performed correctly, rather than forcing automated medical interpretation. The current shape of the project followed.

    The B2B model linked to private insurers — considered in early planning — wasn't completed within the one-month window after the scope shift. Possible extensions remain: rehab-adherence-driven premium discounts, B2B partnerships with fitness centers and rehab clinics, B2C home-training app expansion, gamification.

    Technical growth

    The biggest takeaway was thinking carefully about the line between what AI can technically do and what it can responsibly deliver.
    We hear "is there anything AI can't do these days?" all the time. That perception needs correcting; this project made the difference between humans and AI very visible.

    It was also my first project using LangChain, which gave me hands-on experience building an LLM-based RAG system. Adopting RAGAS for objective measurement was valuable. Going forward, that experience will be central to continuously improving RAG reliability and quality.
