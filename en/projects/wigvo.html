    Overview

    A PSTN-based AI relay platform for real-time bidirectional phone interpretation

    The project started with a story from a startup founder. A foreign employee working in Daegu was being transferred to Seoul and needed to find housing — but couldn't make a single phone call because the language barrier was too high. Translation apps existed, but they fell apart over a phone line. Many people live with that exact wall, so we set out to break it.

    Real-time bidirectional speech translation, long thought to require wideband audio (16 – 24 kHz), is implemented here over the legacy PSTN narrowband (G.711 μ-law, 8 kHz). Echo cancellation and VAD were re-architected as independent components and shipped to production.

    Highlights

    • First approach to deliver real-time bidirectional speech translation over PSTN purely in software (no hardware AEC, no carrier infrastructure)
    • Per-call cost — $0.18/min (33% reduction via architectural optimization)

    Technical Challenge — Why PSTN Is Hard

    Audio quality gap

    PCM16 (16 – 24 kHz) environments assume wideband audio and client-side AEC. PSTN is narrowband G.711 μ-law 8 kHz with 80 – 600 ms variable delay and persistent codec compression noise.

    Echo loop

    The TTS audio reflects back over the PSTN 80 – 600 ms later, re-enters the STT → translation → TTS pipeline, and creates an infinite loop. It happened in 8 of the first 10 test calls. Without the client AEC that wideband apps assume, the loop must be killed in software.

    VAD failure

    OpenAI Server VAD assumes clean wideband audio. PSTN background noise (RMS 50 – 200) is interpreted as ongoing speech, and speech_stopped fires 15 – 72 s late, or never fires at all.

    Architecture — Dual-Session Echo Gating

    The browser client opens a WebSocket to the relay server, which manages two independent Realtime LLM sessions plus the Twilio phone gateway. AudioRouter delegates events via the Strategy pattern to one of three pipelines (V2V, T2V, FullAgent).

    3-layer structure

    • Layer 1 — Transport: Twilio Media Streams (PSTN ↔ G.711 μ-law 8 kHz) + Browser WebSocket (PCM 16 kHz)
    • Layer 2 — Pipeline: AudioRouter delegates events to V2V / T2V / Agent modes via the Strategy pattern
    • Layer 3 — Sessions: Session A (browser → phone) and Session B (phone → browser) hold independent system prompts and a 6-turn sliding context

    Separating STT from translation

    Letting the Realtime API also do translation produced hallucinations — content not present in the original. STT stays on the in-Realtime Whisper-1, but translation is split out to GPT-4o-mini Chat API (temperature=0). Setting context_prune_keep=0 fully blocks Realtime's own translation.

    Stage 1 — Echo Gate (7 iterations)

    Software-only blocking of the TTS-over-PSTN echo loop.

    The breakthrough — Drop vs Replace

    "Dropping" audio reads to Server VAD as a stream interruption, which freezes it. "Replacing" with μ-law silence (0xFF) keeps the stream continuous and VAD reads it as silence as expected. This Drop-vs-Replace idea is the core principle applied identically in the Echo Gate and the VAD path.

    Final Echo Gate — 3 zones

    • Echo window — replace inbound PSTN audio with μ-law silence during TTS playback + 0.5 s jitter margin after TTS ends
    • Dynamic settling — TTS length × 0.3 (clamped 0.5 – 1.5 s) suppresses AGC recovery noise; RMS ≥ 500 still passes as real speech
    • Normal zone — RMS ≥ 150 energy threshold

    7-step iteration

    • Audio Fingerprint (Pearson correlation) — G.711 μ-law non-linear quantization breaks the correlation; didn't work at all
    • Fixed Echo Gate (2.5 s) — removes echo but stalls conversation flow
    • Dynamic Cooldown — gating proportional to TTS length; AGC noise spikes immediately after gate release
    • Silence Injection + RMS + Dynamic Settling + Silero — final design

    Result

    Echo-loop incidence: 8/10 calls in early prototype → 0/148 in production.

    Stage 2 — Independent PSTN VAD architecture

    Problem

    OpenAI Server VAD is a black box; per-frame control inside the echo window isn't possible. RMS thresholds at 150 → 80 → 30 → 20 were tried, but no single threshold is stable on PSTN. Switched to local Silero VAD for an architecture tuned to PSTN.

    2-stage independent filter

    • Stage 1 — RMS Energy Gate: ≥ 500 inside the echo window, ≥ 200 during settling, ≥ 150 normally
    • Stage 2 — Silero VAD: neural classification on frames that pass the energy gate; 8 kHz → 16 kHz zero-order-hold upsampling
    • Asymmetric hysteresis: onset 160 ms (5 frames) / offset 800 ms (25 frames)
    • Min utterance 250 ms, min peak RMS 300 — weak signals rejected as noise

    Result

    speech_stopped latency: 15 – 72 s → 480 ms.

    Stage 3 — 3-stage Whisper-hallucination filter

    When PSTN noise hits Whisper-1, it produces "plausible" text learned from its training data (YouTube, broadcasts) — phrases like "MBC News, this is Lee Deok-young" or "Thanks for watching" leaked into the translation pipeline and went out the recipient's phone in production.

    3-stage pipeline

    • Pre-STT (Stage 0) — Echo Gate + Silence Injection prevents corrupted audio from reaching Whisper at all
    • Post-STT (Stage 1) — Block-list of 51 broadcast-style patterns (29 Korean + 22 English) + a 4-layer text filter combining min-length, silence timeout, repetition, and confidence
    • Post-Translation (Stage 2) — 3-level Guardrail: L1 (pass, 0 ms) · L2 (TTS immediately + background correction, 0 ms) · L3 (block + GPT-4o-mini correction, ~800 ms)

    Result

    Hallucination ingress under 0.3%; on average 0.7 blocked per call (over 148 calls). 95%+ of cases are L1, adding no extra latency.

    Strategy Pattern — 3 communication pipelines

    Refactored the original AudioRouter "God Object" into a thin delegator + 3 independent pipelines via the Strategy pattern (73% code reduction).

    Pipelines

    • VoiceToVoice (V2V) — bidirectional voice translation. Echo Gate + Silence Injection + 3-tier interrupt priority
    • TextToVoice (T2V) — for users with hearing/speech disabilities. Text in → translated AI voice out
    • FullAgent — AI proxy call for phone-averse users. Inherits TextToVoice + Function Calling
    • EchoGateManager — shared echo-prevention logic across all pipelines
    • ChatTranslator — translation for T2V/Agent Session B (GPT-4o-mini)

    Finding

    Signal-correlation echo detection does not work on PSTN. Only directly controlling the echo window and replacing it with silent frames is reliable. The Realtime API's generative behavior is well-suited to STT but unsuitable for translation — splitting out a temperature=0 Chat API improves both accuracy and stability.
