## 7. 이력서용 핵심 스토리 후보

### 스토리 1: "5주 만에 LLM 부하 테스트 풀스택 도구 구축"
- **문제 정의**: vLLM 기반 LLM 서빙 인프라의 성능을 체계적으로 측정하고 최적 설정을 찾는 도구가 부재
- **해결 과정**: FastAPI 백엔드 + Next.js 프론트엔드 + CLI 도구의 3-tier 아키텍처 설계, GPU 모니터링/검증 루프/AI 분석 보고서까지 5주간 45개 커밋으로 구축
- **성과**: 벤치마크 실행 → 실시간 모니터링 → AI 기반 자동 분석 → 인프라 추천까지 원스톱 파이프라인 완성

### 스토리 2: "LLM 기반 벤치마크 자동 분석 시스템"
- **문제 정의**: 벤치마크 결과를 수동으로 해석하는 데 시간 소모
- **해결 과정**: Claude API를 활용한 자동 분석 보고서 생성, thinking 블록 필터링 문제 해결, 국제화 지원 후 간소화
- **성과**: 벤치마크 완료 시 자동으로 성능 병목, 최적화 제안, 인프라 추천이 포함된 AI 분석 보고서 생성

### 스토리 3: "검증 루프(Validation Loop) 시스템 설계"
- **문제 정의**: 부하 테스트 결과의 신뢰성 검증이 수동으로 이루어짐
- **해결 과정**: Docker 로그 수집 모듈, 실시간 검증 루프, 요청/검증 로그 패널 구현 (단일 커밋 5,982줄)
- **성과**: 부하 테스트와 동시에 서버 상태를 실시간 검증하고 이상 탐지 가능

### 스토리 4: "오픈소스 공개 준비 과정"
- **문제 정의**: 내부 도구를 오픈소스로 전환하기 위한 보안/구조 정비 필요
- **해결 과정**: 프로젝트 구조 2회 리팩토링, 보안 점검, CI/CD 구축, README 영문화, 민감정보 제거
- **성과**: GitHub 공개 가능 수준의 코드 품질과 문서화 달성
